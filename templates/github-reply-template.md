# GitHub 外部参与回复模板

## 1. 适用场景

- openclaw/openclaw 的 issues / discussions
- 有明确结构混乱（memory/identity/bootstrap/injection/runtime stability）
- 有机会提供局部清晰化解释

## 2. 不适用场景

- 纯 bug 报告，没有概念层内容
- 涉及项目内部决策或 roadmap
- 已进入多人争论或混战
- 需要讲 FAO 整体理论才能说清

## 3. 回复原则

| 原则 | 执行 |
|------|------|
| 不提 FAO | 不说"FAO 认为"、"我们的方法是" |
| 不替项目表态 | 不说"项目应该"、"官方会" |
| 不抢定义权 | 不说"这就是"、"本质上" |
| 现象语言 | 说"看起来像是"、"可能表现为" |
| 局部澄清 | 只解释一个点，不展开整体 |

## 4. 最小回复骨架

```
【观察】
你提到的 X 问题，看起来像是 Y 机制的表现。

【局部澄清】
在 Z 场景下，这通常表现为...

【边界】
这只是局部观察，不涉及整体架构判断。
```

## 5. 退出提醒

**以下情况不再继续回复**：
- 对方追问超出局部澄清范围
- 讨论升级为争论
- 需要进入建议层（"应该怎么做"）
- 需要替项目表态
- 任何让我需要说"我觉得"而不是"我观察到"的时刻
