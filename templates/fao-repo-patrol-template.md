# FAO 项目巡检输出模板
# 
# 任务性质：内部仓库决策辅助
# 任务边界：仅处理本仓库 PR/Issue/Branch 变更，不处理外部社区观察
# 
# 与外部观察任务的明确区分：
# - 本模板禁止出现：观察期、值得关注、社区动态、思想邻近性、暂不参与
# - 本模板只关注：文件变更、框架一致性、合并决策、下一步动作

## 硬规则

### 规则 1：没读 diff，不得给合并建议
- 仅看标题/body 时，唯一允许输出：
  **fail-closed：信息不足，待读 diff 后重评**
- 违反此规则 = 输出作废

### 规则 2：信息不足时必须 fail-closed
- **禁止输出**: "信息不足，但建议暂缓"
- **必须改成**: "信息不足，fail-closed，下一步动作：获取 X 信息后重评"

### 规则 3：不输出与决策无关的信息
- **删除或禁止**:
  - "活跃度平稳"
  - "总体健康度良好"
  - "值得关注"（但无具体动作）
- **只保留**: 影响判断所需的最小信息集

---

## 必填检查清单

输出前必须确认：

- [ ] 已读 diff（文件级别）
- [ ] 已分类变更类型（core/runtime/mapping/notes/docs/other）
- [ ] 合并建议有具体理由（不是套话）
- [ ] 下一步动作可执行（有明确动词和对象）

## 外部观察语言禁止表

以下表述**禁止**出现在内部巡检输出中：

| 禁止表述 | 替代写法 |
|---------|---------|
| 观察期内容 | (删除，改用变更类型分类) |
| 值得关注的讨论 | (删除，改用具体影响判断) |
| 社区动态 | (删除，这是外部观察任务用语) |
| 思想邻近性 | (删除，这是外部观察任务用语) |
| 暂不参与 | (删除，内部巡检不涉及参与决策) |
| 活跃度平稳 | (删除，与决策无关) |
| 总体健康度 | (删除，与决策无关) |

## 任务边界声明

**本任务是内部仓库决策辅助，不是外部观察任务。**

**本任务只处理：**
- 本仓库 PR / Issue / branch 变更
- 对框架入口和结构的一致性影响
- 是否可合并以及下一步动作

**本任务不处理：**
- 外部项目动态观察
- 社区讨论跟踪
- 思想邻近性分析
- GitHub 外部参与决策

外部观察任务请使用：
- `hermes-agent-observer` (NousResearch Hermes 动态)
- `openclaw-external-observer` (OpenClaw 社区动态)
