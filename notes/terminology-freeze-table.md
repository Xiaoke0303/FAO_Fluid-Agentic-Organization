# FAO 术语冻结表

> **工作底稿 / 白皮书前术语冻结**
>
> 本文是白皮书改写前的工作底稿，不是正式 framework 文件。
> 用途：在正式撰写白皮书正文前，统一术语口径，冻结语种，避免在正文写作中出现中英文混写、口径漂移。

---

## 一、语种冻结规则

### 1.1 主语种
白皮书正文唯一主语种：**中文**

### 1.2 英文保留场景（仅限以下三种）
1. **括注**：首次出现的专业术语后附英文原文，如"真实性合同（Truth Contract）"
2. **锚点说明**：指向具体 framework 文件路径时，如"参见 `TRUTH-CONTRACT.md`"
3. **允许近义表述**：在元动作 crosswalk 等技术映射场景中，标注英文对应词以方便跨平台对照

### 1.3 禁止场景
- 正文中裸写英文术语（如"这是一个 truth status 问题"）
- 中英混写形成半吊子表达（如"这个 task 的 drift 很严重"）
- 在术语表"术语"列中出现英文

## 三、白皮书写作时的术语使用规则

### 3.1 必须遵守
1. **术语列只用中文**：不要在正文章节标题、段落中裸写英文术语
2. **首次出现括注**：专业术语首次出现时，后附英文原文，如"真实性合同（Truth Contract）"
3. **后文只用中文**：首次括注后，全文只使用中文术语
4. **锚点文件保留英文路径**：指向具体 framework 文件时，路径保留英文，如"参见 `TRUTH-CONTRACT.md`"

### 3.2 禁止行为
- ❌ "这是一个 truth status 问题"
- ❌ "任务在 runtime 层漂移了"
- ❌ "需要 fail-closed 处理"
- ❌ "这个 task 的 cost 很高"
- ❌ 中英文混写、中英混词

### 3.3 标记体系使用规则
| 标记 | 使用场景 | 书写格式 |
|------|----------|----------|
| `[verified]` | 已验证内容 | 中文语境写"已验证（`[verified]`）" |
| `[unverified]` | 未闭合内容 | 中文语境写"未闭合（`[unverified]`）" |
| `[inferred]` | 推断内容 | 中文语境写"推断（`[inferred]`）" |
| `[guessed]` | 猜测内容 | 中文语境写"猜测（`[guessed]`）" |
| `[not_completed]` | 未完成内容 | 中文语境写"未完成（`[not_completed]`）" |
